WTS: Maintenance Concerns

  1. Record Locking and Key Assignment need to be atomic operations

    This is a general problem within MGI. We need to ensure that we can do a "select", a test, and an "update" as a single operation when assigning keys or locking records. Once the group finds a general solution, we will need to update WTS code in:

    • TrackRec.py : lock (), unlock ()
    • WTS_DB_Object.py : allocate_Key ()

  2. Names of help files are hard-coded

    We are tying hard-coded filenames in TrackRec.py (HELP_FILES) to files which are expected to be in the file system. If we need to change one of these filenames, then we need to change the value in:

    • TrackRec.py : HELP_FILES
    • wts.screens.py : gen_Help_Screens ()
